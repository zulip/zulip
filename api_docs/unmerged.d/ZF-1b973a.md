**Feature level ZF-1b973a**

* [`GET /events`](/api/get-events): The `avatar_source` field in
  [`realm_user`](/api/get-events#realm_user-update) events now includes
  a new value `"I"` (inaccessible user avatar), sent when a user is
  spam-deactivated. Clients can continue to use `avatar_url` to display
  the correct avatar, rather than interpreting `avatar_source`.
