* [`POST /users/me/subscriptions`](/api/subscribe): Added
  `push_notifications_enabled` parameter that controls whether mobile push
  notifications are enabled by default for new subscribers of the channel.
  Only organization administrators can set this to `true`.

* [`POST /channels/create`](/api/create-channel): Added
  `push_notifications_enabled` parameter that controls whether mobile push
  notifications are enabled by default for new subscribers of the channel.
  Only organization administrators can set this to `true`.

* [`PATCH /streams/{stream_id}`](/api/update-stream): Added
  `push_notifications_enabled` parameter that controls whether mobile push
  notifications are enabled by default for new subscribers of the channel.
  Only organization administrators can set this to `true`.

* [`POST /register`](/api/register-queue), [`GET /events`](/api/get-events),
  [`GET /streams`](/api/get-streams),
  [`GET /streams/{stream_id}`](/api/get-stream-by-id): Added
  `push_notifications_enabled` field to channel objects, indicating whether
  mobile push notifications are enabled by default for new subscribers. When
  `true`, new subscriptions to the channel will have mobile push notifications
  enabled by default, overriding the subscriber's account-level default.
