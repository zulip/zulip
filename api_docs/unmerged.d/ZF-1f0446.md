- `PATCH /realm`, [`POST /register`](/api/register-queue),
  [`GET /events`](/api/get-events):
  Added `realm_topic_resolution_message_requirement` realm setting, which
  controls whether a message is required when resolving topics. Valid values
  are `not_requested` (default), `optional`, and `required`.
- [`POST /messages`](/api/send-message):
  Added `then_resolve_topic` boolean parameter for channel messages to atomically
  resolve the topic after sending a message.
