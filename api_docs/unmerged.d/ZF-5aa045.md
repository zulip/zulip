**Feature level ZF-5aa045**

* [`GET /messages`](/api/get-messages): The `channels` operator (and its
  legacy alias `streams`) now accepts a JSON array of channel IDs as its
  operand, in addition to the string operands `"public"`, `"web-public"`,
  and `"archived"`. For example, `{"operator": "channels", "operand": [1, 2, 3]}`
  searches messages across the channels with IDs 1, 2, and 3. The narrow is
  rejected with an error if the user lacks access to any listed channel or if
  any channel does not exist. When combined with `is:muted`, an explicitly
  listed channel is not excluded merely because the channel itself is muted;
  only topic-level mutes within the listed channels apply.
