* [`POST /register`](/api/register-queue): Added `push_accounts`
  field to response.
* [`GET /events`](/api/get-events): A `push_account` event with
  `op: "update"` is sent to clients when registration status of
  the account changes from `"pending"` to `"active"`, `op: "remove"`
  is sent in the case of registration failure.
* [`POST /mobile_push/register`](/api/register-push-account): Added
  an endpoint to register an account on an install of the app
  to receive end-to-end encrypted mobile push notifications.
