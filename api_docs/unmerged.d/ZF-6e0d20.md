* [`POST /register`](/api/register-queue): Added the
  `long_term_idle_reactivation` client capability. When a returning
  long-term-idle user registers with this capability, the server
  reactivates the account in the background instead of synchronously,
  and the response contains only `queue_id` and `last_event_id` plus a
  new `long_term_idle_reactivating` boolean set to `true`.
* [`GET /events`](/api/get-events): Added a `long_term_idle` event,
  sent to a client that registered with the
  `long_term_idle_reactivation` capability once the returning user's
  background reactivation finishes, so the client can reload into the
  loaded account.
