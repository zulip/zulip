**Changes**: The `GET /users` endpoint now supports an optional `include_avatar_source` query parameter. When set to `true` (available for realm administrators only), the response will include an `avatar_source` field in each user object. This field indicates the avatar data source type: `"G"` (Gravatar), `"J"` (Jdenticon), or `"U"` (uploaded by user). It is present whenever `avatar_url` is present in the response.
